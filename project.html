2048 game using AI

The 2048 game is a single-player game.In the project, we implemented the Minimax Algorithm, the Expectimax Algorithm and Reinforcement Learning to create agents that can master the game. Following were the parameters used in the project:
Performance Measure:Tile with the maximum value on the board
Environment: 2048 game board of 16 squares in a 4x4 configuration.
Actuators: Virtual keyboard events ‘Up’, ‘Down’, ‘Left’, ‘Right’.
Sensors: Interaction with game logic.
Essentially, the projects involves the implemenation and comparative analysis of the three algorithms.
Implementation language: Python
Libraries: tkinter, keras, numpy, random

Wine quality prediction

Implemented random forest and decision tree algorithms in machine learning to predict the quality of wine. The goal of the project was to determine white wine quality based on its chemical properties. The dataset was downloaded from UCI and it contains 11 attributes and the 12th column is quality which is regarded as the label or target data. Quality is graded from 0 to 10, where 0 is the worst and 10 is the best quality. Based on the quality scores we classify wines as Good, Bad and Normal ones. Essentially, the accuracy of random forest algorithm and decision tree algorithm had been compared.
Dataset: https://archive.ics.uci.edu/ml/datasets/wine+quality
Implementation language: Python
Libraries: sklearn, numpy, seaborn, matplotlib, graphviz, pandas.

Database Implementation of IPL(Indian Premiere League)

The domain of this database was based on Indian Premier League (IPL). IPL, is a professional Twenty20 cricket league in India, which is contested during April and May of every year by 8 teams representing Indian cities and some states.Each team plays each other twice in a home-and-away round-robin format. It is the most-attended cricket league in the world. The data for a few tables was downloaded online, while some of the tables was populated by me.
Dataset: https://www.kaggle.com/harsha547/indian-premier-league-csv-dataset/version/2
Database: PostgreSQL

Analysis of QuickSort, RadixSort, TimSort

Implemented QuickSort with naive pivot selection, random pivot selection and median of medians with group 5 and group 7. To scrutinize the algorithms thoroughly, different input lists are generated as: Unique random integers list, Few unique random integers list(may contain duplicate integers), Nearly sorted (first half sorted, second half random), Nearly sorted( second half sorted, first half random),Reverse integers list (sorted in decreasing order).
Implementation Language: Python
Input sizes: 1000,10000,100000,1000000.
Libraries: random
